Question 1: Difference b/w HTML & HTML5?

Full form of HTML is Hyper Text Markup Language. HTML is use to design web pages. HTML5 is the 5th and latest verion of HTML. It has updated features which suppports multimedia, tags and elements and new APIs. Here are major differences mentioned below.

Basis HTML HTML 5
html structure Mostly uses < div> and non-semantic tags Uses semantic tags like < section>, < article>, < header>, < footer>, etc.
media support use third party plugin like: Adobe Flash, Microsoft Silverlight, and Google Gears JavaScript runs directly on the browser through the JS Web Worker API
Graphics and Animation Not supported directly; needs plugins < canvas> and < svg> allow direct drawing and animations
Form Controls Basic input types: text, radio, checkbox, etc. New input types: date, email, tel, url, search, range, color, etc.
2-D shapes drawing Shapes like circles, rectangles, triangles, etc. are not possible to draw in HTML Shapes like circles, rectangles, triangles, etc. are easy to draw in HTML5
Mobile friendliness HTML is less mobile-friendly HTML5 is more mobile-friendly
Drag and Drop It does not support drag and drop effects It supports the drag and drops effect
Geolocation Support Not supported Built-in Geolocation API
Error Handling Inconsistent between browsers Unified error handling rules across browsers

Question 2: What are the additional tags used in HTML5?

Here is new semantic tag

  • < article>: Represents self-contained content, such as a blog post or a news story, that could be distributed independently.
  • < aside>: Defines content that is tangentially related to the main content around it, often presented as a sidebar.
  • < details>: Creates a disclosure widget from which the user can obtain additional information or controls on demand.
  • < figcaption>: Provides a caption or legend for a < figure> element.
  • < figure>: Encapsulates self-contained content, often an image, diagram, or code snippet, that is referenced from the main flow of the document.
  • < footer>: Represents the footer of a document or section, typically containing information like author details, copyright, and related links.
  • < header>: Represents a container for introductory content or a set of navigational links for a document or section.
  • < main>: Specifies the main or dominant content of the document.
  • < mark>: Defines text that is highlighted or marked for reference purposes.
  • < nav>: Declares a section of navigation links, intended for the major navigation blocks of a site.
  • < section>: Defines a generic section of a document, such as chapters, headers, or any other thematic grouping of content.
  • < summary>: Defines a visible heading for the < details> element.
  • < time>: Represents a specific time and/or date in a machine-readable format.
  • < wbr>: Represents a "word break opportunity", suggesting a spot where a browser can optionally break a long word or URL to improve layout on narrow screens.

Here is new Media tag

  • < audio>: Used to embed sound content or audio streams.
  • < canvas>: Provides a region for drawing dynamic bitmap graphics and animations using JavaScript.
  • < embed>: Defines a container for external applications or interactive content (this was supported by most browsers previously, but made part of the HTML5 specification).
  • < source>: Specifies multiple media resources (different file formats) for < audio>, < video>, and < picture> elements.
  • < track>: Defines text tracks (like subtitles or captions) for < audio> and < video> elements.
  • < video>: Used to embed video content, such as movie clips.
  • < picture>: A container for multiple < img> sources, allowing browsers to select the best image for the current viewport/device characteristics.

Here is new Form tag

  • < datalist>: Provides a list of predefined options for an < input> element, creating a combobox experience.
  • < dialog>: Defines a dialog box or subwindow.
  • < meter>: Represents a scalar measurement within a known range (a gauge).
  • < output>: Used as a container to display the result of a calculation performed by a script.
  • < progress>: Represents the completion progress of a task, typically a progress bar for an ongoing process.

Here is new Input tag

  • Color: The color type is used for input fields that should contain a color
  • Date: The date type allows the user to select a date
  • email:The datetime type allows the user to select a date and time
  • Month: The month type allows the user to select a month and year
  • Number: The number type is used for input fields that should contain a numeric value.
  • Range: The range type is used for input fields that should contain a value from a range of numbers.
  • Search: The search type is used for search fields
  • Time: The time type allows the user to select a time
  • Url: The url type is used for input fields that should contain a URL address.
  • week: The week type allows the user to select a week and year
  • Autocomplete: Specifies whether an < input> element should have autocomplete enabled
  • Accept: Specifies the types of files that the server accepts (only for type="file"). We can attach all types of doc, image and zip file
  • Autofocus: Specifies that an < input> element should automatically get focus when the page loads
  • List: Refers to a < datalist> element that contains pre-defined options for an < input> element
